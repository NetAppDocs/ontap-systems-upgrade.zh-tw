= 使用AFF EsceeA900控制器模組和NVRAM模組來開機節點2
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
節點2搭配AFF EsceA900控制器模組和NVRAM模組、現在已準備好升級。透過交換控制器模組和NVRAM模組、將AFF 某個處理器從某個部分升級至AFF 某個部分、只需移動主控台和管理連線即可。本節提供使用AFF E4A900控制器模組和NVRAM模組來開機節點2所需的步驟。

.步驟
. [[boot_node2_step1]]如果此組態正在使用NetApp儲存加密（NSE）、則必須將「setenv bootarg.storageEncryption.support'命令設定為「true」、並將「kmip.init.maxwait`變數」設定為「Off」、以避免在節點2組態載入後發生開機迴圈：
+
「etenv bootarg.storageencryption。支援true」

+
「kmip.init.maxwait關」

. 將節點開機至「boot_menu」：
+
Boot_ONTAP功能表

. 節點會在開機功能表停止。輸入「22/7」、然後選取隱藏選項「boot_after控制器置換」。若要將AFF EsceA700 node1磁碟重新指派至AFF EsceA900 node1、請在提示字元下輸入節點2的實際節點名稱。請參考下列範例：
+

[NOTE]
====
在前述主控台輸出範例中、ONTAP 如果系統使用進階磁碟分割（ADP）磁碟、則會提示您輸入合作夥伴節點名稱。

上例所示的系統ID為範例ID。您要升級之節點的實際系統ID會有所不同。

在提示輸入節點名稱與登入提示之間、節點會重新開機數次、以還原環境變數、更新系統卡上的韌體、以及進行其他ONTAP 的更新。
